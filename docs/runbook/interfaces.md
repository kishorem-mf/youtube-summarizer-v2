# Runbook — YouTube Summarizer v2 Interfaces

## Starting the App

### Prerequisites
- Python venv activated
- `.env` file present with all keys (AWS, Anthropic, YouTube, Apify)
- `corporate-ca-bundle.pem` present (auto-generated on first run if missing)

### Start command
```bash
# From project root
venv\Scripts\python app.py
```

App starts on **http://127.0.0.1:8051**

To use a different port:
```bash
PORT=8080 venv\Scripts\python app.py
```

### Stop
`Ctrl+C` in the terminal running the app.

## Interface 1 — YouTube Summary Watch
**URL:** `http://127.0.0.1:8051`

The main app for searching YouTube and generating AI summaries.

### How to use
| Step | Action |
|------|--------|
| 1 | Select one or more search terms from the grouped checkboxes, or type a custom term in the **Custom terms** box |
| 2 | Set **Max results**, **Uploaded**, **Sort by**, and **Detail** level |
| 3 | Click **Run Search** — results appear grouped by term |
| 4 | Click **View Transcript & Summary** on any video card to open the video page |
| 5 | On the video page, choose a detail level and click **Get Summary** |

### Detail levels
| Level | Output |
|-------|--------|
| Low   | TL;DR + 3–5 bullet points |
| Medium | Structured summary with key sections |
| High  | Full detailed breakdown |

### Notes
- First request for a video+detail calls the LLM (~10–15s)
- Second request for the same video+detail returns instantly from DynamoDB cache
- A digest markdown file is saved to `outputs/digest_YYYY-MM-DD.md` after each search run

## Interface 2 — Search Terms Manager
**URL:** `http://127.0.0.1:8051/terms`

Manage the search terms used by the main app without touching code.

### How to use

#### Add a term to an existing group
1. Under **Add Search Term**, select a group from the dropdown
2. Type the new term (e.g. `vector databases`)
3. Click **Add Term** — the term appears as a chip in that group immediately

#### Add a new group
1. Under **Add New Group**, enter a group name (e.g. `AI Tools`)
2. Optionally enter a **context suffix** — this is appended to every search query in the group to keep results on-topic (e.g. `machine learning` turns `LangChain` into `LangChain machine learning`)
3. Click **Create Group**

#### Delete a term
- Click the **✕** button on any term chip

#### Edit group context suffix
- Each group card shows a **Context suffix** field — edit and click **Save**

#### Delete a group
- Click **Delete group** on the group card — removes the group and all its terms (confirmation prompt shown)

### How changes take effect
- Changes write to `search_terms.json` immediately
- The main app reads from `search_terms.json` on every search — **no restart needed**
- `search_terms.json` is gitignored so personal term changes stay local

## Interface 3 — DynamoDB Explorer
**URL:** `http://127.0.0.1:8051/dynamo`

Query and inspect the DynamoDB cache without writing code or opening the AWS Console.

### Query types

| Query | Fields | Use case |
|-------|--------|----------|
| **Q1 — Get by Video ID + Detail** | Video ID, Detail level | Look up a specific cached summary |
| **Q2 — All detail levels for a Video** | Video ID | See all cached summaries (low/medium/high) for one video |
| **Q3 — Browse recent items** | Limit (1–100) | See latest cached entries, sorted newest first |
| **Q4 — Delete cache entry** | Video ID, Detail level | Invalidate a stale or incorrect summary |

### How to use

#### Look up a specific summary (Q1)
1. Select **Q1 — Get by Video ID + Detail**
2. Enter the YouTube video ID (e.g. `dQw4w9WgXcQ` — the part after `?v=` in the URL)
3. Select detail level (`low`, `medium`, `high`)
4. Click **Run Query**

#### Browse recent entries (Q3)
1. Select **Q3 — Browse recent items**
2. Set **Limit** (default 20, max 100)
3. Click **Run Query** — results sorted newest first

#### View full summary
- In the results table, the **Summary** column shows the first 80 characters
- Click the `▼` arrow to expand the full formatted summary inline
- Click again to collapse

#### Delete a cache entry (Q4)
- Select **Q4 — Delete cache entry**, enter Video ID + Detail, click **Delete Entry**
- Confirmation prompt shown before deletion
- Use this to force a fresh LLM call on the next request for that video

### Columns in results table
| Column | Description |
|--------|-------------|
| Video ID | Clickable link to the YouTube video |
| Detail | Summary level (low / medium / high) |
| Title | Video title |
| Channel | YouTube channel name |
| Search Term | The search query that surfaced this video |
| Tags | Up to 6 LLM-generated topic tags |
| Searched On | Date the summary was first cached |
| Source Type | `llm` = generated by LLM · `cache` = served from cache |
| Usage | Number of times this summary has been requested |
| Words | Word count of the original transcript |
| Language | Transcript language code |
| Summary | Click `▼` to expand full text |

## Troubleshooting

| Problem | Likely cause | Fix |
|---------|-------------|-----|
| App won't start | Missing `.env` key | Check all vars in `.env` are set |
| SSL error on AWS calls | Corporate proxy cert | Ensure `AWS_CA_BUNDLE` points to `corporate-ca-bundle.pem` |
| Summary takes too long | LLM cold start | Normal on first request; subsequent requests use cache |
| Cache not updating | DynamoDB connection issue | Check AWS credentials and region in `.env` |
| New search term not appearing in main app | `search_terms.json` not saved | Check `/terms` for a flash error message |
